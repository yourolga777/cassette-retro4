ЧАСТЬ 2. РАБОТА НАД TELEGRAM-БОТОМ
ЭТАП 1. БАЗОВАЯ НАСТРОЙКА БОТА
1.1. Зарегистрировать бота @acassette_bot у @BotFather, получить токен
1.2. Настроить вебхук (или long polling) на ваш сервер
1.3. Создать файл bot.py с подключением к токену
1.4. Реализовать команду /start с приветствием и описанием возможностей
1.5. Сделать проверку: бот работает только для списка разрешённых Telegram ID (админов)
1.6. Настроить логирование всех действий бота в отдельный файл

ЭТАП 2. УВЕДОМЛЕНИЯ О НОВЫХ ЗАКАЗАХ
2.1. В коде сайта при создании заказа вызвать функцию send_notification_to_bot
2.2. Формат уведомления: номер заказа, сумма, способ оплаты, состав заказа (кратко)
2.3. Добавить в уведомление кнопки «Подтвердить заказ» и «Отклонить заказ»
2.4. При нажатии «Подтвердить» менять статус заказа в БД и уведомлять клиента (если есть контакт)
2.5. При нажатии «Отклонить» отправлять клиенту причину (заполняет менеджер в боте)
2.6. Сделать отдельное уведомление для заказов с наличными (жирным шрифтом или маркером)

ЭТАП 3. УПРАВЛЕНИЕ ЗАКАЗАМИ ЧЕРЕЗ БОТА
3.1. Команда /pending - показать список неподтверждённых заказов с кнопками
3.2. Команда /orders_today - все заказы за сегодня
3.3. Команда /order 123 - детальная информация по конкретному заказу
3.4. Команда /mark_paid 123 - вручную отметить заказ как оплаченный (для наличных)
3.5. Команда /status 123 - изменить статус заказа (в обработке, отправлен, доставлен)
3.6. Команда /cancel_order 123 - отменить заказ с возможностью написать причину
3.7. Сделать клавиатуру с часто используемыми командами (меню)

ЭТАП 4. УПРАВЛЕНИЕ КАТАЛОГОМ ЧЕРЕЗ БОТА
4.1. Команда /catalog - показать список товаров (ID, название, цена, в наличии)
4.2. Команда /add_product - запустить диалог для добавления товара
4.3. В диалоге запросить: название, исполнитель, цену, описание, количество, фото
4.4. После подтверждения создать товар в БД сайта
4.5. Команда /edit_product ID - изменить цену или количество на складе
4.6. Команда /delete_product ID - удалить товар (с подтверждением)
4.7. Команда /stock_low - показать товары с остатком меньше 3 штук

ЭТАП 5. УПРАВЛЕНИЕ БЛОГОМ ЧЕРЕЗ БОТА
5.1. Команда /new_post - запустить диалог создания поста
5.2. Запросить заголовок, текст, теги (через запятую)
5.3. Показать предпросмотр поста и запросить подтверждение
5.4. Сохранить пост в БД сайта со статусом «черновик»
5.5. Команда /posts_list - показать последние 5 постов из блога с ID
5.6. Команда /publish_post ID - опубликовать черновик на сайте
5.7. Команда /edit_post ID - изменить заголовок или текст существующего поста

ЭТАП 6. ПУБЛИКАЦИЯ ПОСТОВ В TELEGRAM-КАНАЛ
6.1. Попросить пользователя ввести username канала (например @acassette_news)
6.2. Добавить бота в канал как администратора
6.3. Команда /post_to_tg ID - опубликовать существующий пост сайта в канал
6.4. Формат публикации: заголовок + первые 200 символов текста + ссылка «Читать полностью на сайте»
6.5. Добавить к посту в канале кнопку «Перейти к товарам» или «Оставить отзыв»
6.6. Команда /auto_publish_on - включить автоматическую публикацию новых постов в канал
6.7. Команда /auto_publish_off - отключить автоматическую публикацию

ЭТАП 7. ДОПОЛНИТЕЛЬНЫЕ ФУНКЦИИ БОТА
7.1. Команда /stats - показать статистику: заказов сегодня, выручка за неделю, топ-3 товара
7.2. Команда /feedback - диалог для отправки предложений по улучшению бота (админу в личку)
7.3. Команда /broadcast - отправить сообщение всем подписанным клиентам (с осторожностью)
7.4. Сделать обработку фото чеков: клиент кидает фото → бот пересылает админу с кнопкой «Подтвердить оплату»
7.5. Добавить возможность отвечать клиенту прямо из бота (если у клиента есть Telegram)
7.6. Команда /help - подробный список всех команд с описанием
7.7. Ежедневный автоматический отчёт в 10 утра: количество новых заказов за сутки

ЭТАП 8. БЕЗОПАСНОСТЬ И НАДЁЖНОСТЬ БОТА

8.7. Создать резервную копию конфигурации бота (команды, клавиатуры)
